# AnomalyDetection

Программа решает задачу поиска аномалий в выборках объектов (каждый объект задан одним числовым параметром), то есть задачу обнаружения и описания «ненормального» поведения некоторой выбранной пользователем части объектов по отношению ко всей выборке.
Процесс работы с программой заключается во взаимодействии пользователя с графическим интерфейсом программы и файловой системой операционной системы.
Программа реализует алгоритм поиска аномалий на основе быстрого дискретного вейвлет-преобразования с использованием статистических критерий Фишера и Кохрана-Кокса для средних значений и дисперсий.
Программа получает на входе набор объектов, описанных своим числовым параметром (дробное число со знаком) и проверяет, содержит ли указанное пользователем окно обнаружения аномалии (как кратковременные, так и длительные). Результаты поиска можно сохранить в файл.

Условия эксплуатации: 
* объём ПЗУ не менее 10 Мб;
* объём ОЗУ не менее 2 Гб;
* операционная система Windows XP или старше;
* наличие установленной библиотеки .NET Framework от Microsoft версии не ниже 4.7.2.

Использование программного средства:
* запуск программы осуществляется с помощью исполняемого файла «AnomalyDetectionApplication.exe»;
* программа представляет из себя визуальный интерфейс на базе технологии WPF с полями для ввода и кнопками для управления;
* кнопка «Загрузить из файла» открывает «Проводник Windows» и предлагает пользователю выбрать файл с данными для загрузки, обрабатывает его и осуществляет отрисовку данных в виде точек на плоскости и первичную установку окон сравнения и обнаружения;
* блок «Параметры» позволяет выбрать вейвлет, использующийся для поиска аномалий, установить чувствительность детектора аномалий и определить границы окон сравнения и обнаружения
* кнопка «Искать аномалии» запускает процесс поиска аномалий в окне обнаружения на основе данных из окна сравнения;
* блоки «Статистические критерии» и «Результаты поиска аномалий» позволяют изучить результаты выполнения алгоритма;
* кнопка «Сохранить в файл» открывает «Проводник Windows» и предлагает пользователю сохранить результат поиска в файл;
* кнопка «Выход» закрывает окно программы;
* сообщения об ошибках выводятся в диалоговом окне, сообщения содержат соответствующие инструкции.

Данные:
* входные данные должны быть представлены в виде файла с расширением .txt следующего формата:
каждый объект находится на новой строке файла и представляется дробным числом (разделитель зависит от версии ОС) со знаком; 
* выходные данные представляются в виде файла с расширением .txt следующего формата:
блок результатов:
Results:
Далее каждый статистический критерий представлен в следующем виде:
Source:<Название_критерия>
Result:<Тип_результата_поиска>
Value<Значение_статистического_критерия>
Limit:<Порог_статистического_критерия>
блок параметров выборки:
Comparison window:[<Левая_граница_окна_сравнения>,<Правая_граница_окна_сравнения>)
Detection window:[ <Левая_граница_окна_обнаружения>,<Правая_граница_окна_обнаружения>)
Sensivity:<Коэффициент_чувствительности_детектора_аномалий_(1-норма)>
блок данных:
Data
<Все_объекты_в_исходном_виде>
